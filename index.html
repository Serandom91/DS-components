DS Components

Generic, client-agnostic UI component library. Each component connects to tokens.css — override with a client theme file to adapt the full system to any brand.

🎨

3-layer token system

Primitives → raw values (colors, spacing, radii) · Semantics → brand, text, background, feedback · Components → button, input, card, badge tokens.
To theme for a client: create [client]-tokens.css and override the semantic layer.
